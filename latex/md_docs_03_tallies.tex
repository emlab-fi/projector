The tallies are saved into a {\ttfamily .csv} file at location specified in the configuration files.\hypertarget{md_docs_03_tallies_autotoc_md35}{}\doxysection{Uniform mesh tallies}\label{md_docs_03_tallies_autotoc_md35}
These tallies are counted in a uniform grid. Each quantity is then tallied in each cell of the grid.\hypertarget{md_docs_03_tallies_autotoc_md36}{}\doxysubsection{Algorithm for tallying a single particle track (ie, photon flux)}\label{md_docs_03_tallies_autotoc_md36}
For each segment of particle history, do the following\+:


\begin{DoxyEnumerate}
\item determine whether the segment actually goes through the tally space
\item calculate all grid crossing points of the segment
\item for each grid cross point\+:
\begin{DoxyEnumerate}
\item determine corresponding tally cell coordinates
\item tally the requested value
\end{DoxyEnumerate}
\end{DoxyEnumerate}

This should be linear per each segment. Worst case the segment hits at most the number of cells on the longest grid dimension.

This should also be able to be done on all mesh tallies at once, no need to sequentially process each tally.\hypertarget{md_docs_03_tallies_autotoc_md37}{}\doxysubsection{Scores}\label{md_docs_03_tallies_autotoc_md37}
The score is the physical quantity to evualate. Currently supported scores are\+:


\begin{DoxyItemize}
\item Photon flux
\item Number of reactions
\item Average energy
\item Deposited energy 
\end{DoxyItemize}
