\hypertarget{md_docs_00_user_guide_autotoc_md4}{}\doxysection{Installation}\label{md_docs_00_user_guide_autotoc_md4}
Currently we do not provide pre-\/built executables for Projector. The recommended way to install Projector is to build it from either latest development branch or from the main release branch.


\begin{DoxyEnumerate}
\item Download source of desired version
\item Compile the core component
\item Either move the built executable to a directory in {\ttfamily PATH} so that it is globally accessible or use it directly from the build folder.
\item Download \href{https://nucleardata.lanl.gov/ace/eprdata14/}{\texttt{ eprdata14}} ACE files
\item Create an environment variable called {\ttfamily PROJECTOR\+\_\+\+ACE\+\_\+\+XSDIR} which contains path to the {\ttfamily xsdir} file of the {\ttfamily eprdata14} library.
\begin{DoxyItemize}
\item If you skip this step, you need to provide the path to the library every time you launch Projector executable.
\end{DoxyItemize}
\end{DoxyEnumerate}\hypertarget{md_docs_00_user_guide_autotoc_md5}{}\doxysection{Building from source}\label{md_docs_00_user_guide_autotoc_md5}
The build for Projector is quite selfcontained. The only dependencies are {\ttfamily CMake 3.\+12} or newer and C++ compiler supporting C++20. The build is currently only tested on {\ttfamily GCC 13.\+2} and {\ttfamily CMake 3.\+27}, on {\ttfamily Ubuntu 23.\+10}\hypertarget{md_docs_00_user_guide_autotoc_md6}{}\doxysubsection{Building on Linux}\label{md_docs_00_user_guide_autotoc_md6}

Not yet tested, but should work.\hypertarget{md_docs_00_user_guide_autotoc_md8}{}\doxysection{General workflow}\label{md_docs_00_user_guide_autotoc_md8}
The general workflow for simulating a problem in Projector is as follows. Some steps are not yet fully implemented and can change between releases.


\begin{DoxyEnumerate}
\item Define problem in input json files -\/ see \mbox{\hyperlink{md_docs_06_input_format}{input format}} -\/ recommended order is\+:
\begin{DoxyEnumerate}
\item Main configuration file
\item Materials
\item Geometry
\item Tallies
\end{DoxyEnumerate}
\item Launch projector\+\_\+core executable -\/ this step can take a long time!
\item Examine results using tooling -\/ most tooling is currently missing and is not implemented well.
\end{DoxyEnumerate}\hypertarget{md_docs_00_user_guide_autotoc_md9}{}\doxysection{Launching projector}\label{md_docs_00_user_guide_autotoc_md9}
The main Projector executable is called {\ttfamily projector\+\_\+core}. It can run the simulation, validate input files and create data for plots of the geometry. It takes several arguments and options, described below.


\begin{DoxyCode}{0}
\DoxyCodeLine{Usage: .projector\_core [OPTIONS] SUBCOMMAND INPUT}
\DoxyCodeLine{}
\DoxyCodeLine{Positionals:}
\DoxyCodeLine{  INPUT    Input JSON file [REQUIRED]-\/ the main configuration JSON file}
\DoxyCodeLine{}
\DoxyCodeLine{Options:}
\DoxyCodeLine{  -\/h,-\/-\/help                   print help message and exit}
\DoxyCodeLine{  -\/a,-\/-\/ace\_data               path to eprdata14 xsdir -\/ can ommit if provided in PROJECTOR\_ACE\_XSDIR env variable}
\DoxyCodeLine{  -\/t,-\/-\/thread\_count INT       max threads to use}
\DoxyCodeLine{}
\DoxyCodeLine{Subcommands:}
\DoxyCodeLine{  run                         run simulation from input files}
\DoxyCodeLine{  validate                    validates input files}
\DoxyCodeLine{  visualize                   plot input geometry}

\end{DoxyCode}


The visualization subcommand has several more options\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{-\/s,-\/-\/slice      [REQUIRED]  slice plane -\/ must be one of x,y,z}
\DoxyCodeLine{-\/c,-\/-\/center                 center point of the slice, default value is 0.0}
\DoxyCodeLine{-\/r,-\/-\/resolution [REQUIRED]  output resolution, must be two positive integer numbers}
\DoxyCodeLine{-\/o,-\/-\/output                 output .csv file, default value is 'slice\_out.csv'}

\end{DoxyCode}

Future goal of Projector is to provide a set of scripts in Python for easier work with the main executable and for post-\/processing/visualization of tally results. The beggings of the tooling implementation are currently in the {\ttfamily tooling} folder of the repository.
